# DYE MASTER 인트로 힌트 연출 개선 계획 (plan1.md)

## 1. 작업 목적
현재 게임(추측 단계) 진입 시 보여주는 슬라이더 왕복 힌트 애니메이션이 너무 크고 급격하게 변화하여 플레이어에게 시각적 혼란(이미지가 확 바뀌는 느낌)을 줍니다. 이를 1초(1000ms) 동안 작고 부드럽게, 그리고 3개의 슬라이더가 동시에 움직이는 것이 아니라 약간의 시간차(Delay)를 두고 파도 타듯 개별적으로 살짝씩 움직이게 개선하여 "각각의 슬라이더를 이렇게 위아래로 조작할 수 있다"는 개별 조작 힌트를 명확히 제공합니다.

## 2. 핵심 기능
- **연출 시간 연장**: `shuffleDuration`을 600ms에서 1000ms(1초)로 증가시킵니다.
- **개별 파동(Wave) 분리**: 
  - 진행도 `progress`(0~1)에 따라 각 슬라이더의 움직임 구간을 엇갈리게 나눕니다.
  - H 슬라이더: `progress` 0.0 ~ 0.7 구간에서 사인파 왕복. (진폭 20도)
  - S 슬라이더: `progress` 0.15 ~ 0.85 구간에서 사인파 왕복. (진폭 15)
  - L 슬라이더: `progress` 0.3 ~ 1.0 구간에서 사인파 왕복. (진폭 15)
- **변동 폭(진폭) 축소**: 전체 화면 톤이 크게 요동치지 않도록 변위폭을 기존(45, 20)에서 절반 수준(20, 15)으로 줄여 미세한 들썩임 수준으로 안정화시킵니다.

## 3. 입력과 출력
- **입력**: 1초 동안 렌더링 프레임 단위의 `progress` 값.
- **출력**: H, S, L 슬라이더가 각각 순차적으로 부드럽게 들썩였다가 본래 위치로 수렴하는 시각적 애니메이션.

## 4. 파일 구조
- [GameView.js](file:///e:/AI/DYE_MASTER/colorFinder/src/ui/GameView.js): `doShuffle` 함수 내부의 애니메이션 파형 계산식 전면 교체.

## 5. 핵심 모듈
- **Sequential Wave Formulator**: `calcWave(progress, start, end)` 헬퍼 함수를 통해 각 슬라이더별로 독립된 위상과 주기를 갖는 곡선 생성.

## 6. 실행 흐름
1. `transitionToGuess()` 진입 시 1000ms 지속 셔플 시작.
2. 매 프레임마다 `calcWave`가 각각 H, S, L의 움직임 비율을 반환.
3. 각 비율에 맞춰 미세한 진폭(20, 15)을 오프셋에 더하여 부드럽게 스크롤.
4. 애니메이션 도중 화면 터치 발생 시 즉각 중단하고 수동 조작 인계.

## 7. 에러 처리
- `progress`가 할당된 `start~end` 구간을 벗어났을 때 0을 정확히 반환하여 슬라이더가 초기 오프셋 좌표에 안정적으로 머무르도록 예외 처리.

## 8. 테스트 전략
- 추측 화면 진입 시, 왼쪽부터 오른쪽으로 "스르륵, 스르륵, 스르륵" 물결치듯 부드럽게 움직이는지 시각 검토.
- 화면 전체 색상이 어지럽게 확 바뀌지 않고 조작 힌트로서의 역할만 수행하는지 느낌 점검.

## 9. 완료 기준
- 1초 동안 슬라이더들이 개별적으로 작게 움직이는 힌트 연출 구현 완료.
