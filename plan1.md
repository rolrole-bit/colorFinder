# 작업 목적
스크롤바 생김 현상 수정 (중간 결과 화면)

# 핵심 기능
`.split-screen-result` 컨테이너에 스크롤바가 생기지 않도록 `overflow` 속성 조정

# 입력과 출력
입력: 스크롤바가 생기는 결과 화면
출력: 스크롤바가 없는 깔끔한 핏의 화면

# 파일 구조
- `src/index.css` 수정

# 핵심 모듈
- CSS: `.split-screen-result` 클래스

# 실행 흐름
- CSS 업데이트 반영으로 화면 진입 시 오버플로우가 숨겨짐. (블러 처리용 가상 컨테이너가 120% 사이즈라 발생했던 스크롤을 방지)

# 에러 처리
- N/A (CSS 레이아웃 처리)

# 테스트 전략
- 스크린 확인을 통한 UI 정상 노출 및 스크롤바 제거 확인

# 완료 기준
- `.split-screen-result`에 `overflow: hidden`이 정상 반영되었는지 확인.
