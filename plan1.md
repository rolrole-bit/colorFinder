# DYE MASTER H 슬라이더 보색 피킹 오류 해결 계획 (plan1.md)

## 1. 작업 목적
현재 H(색조) 슬라이더 조작 시, 화면 기준선에 위치한 색상과 실제 피킹되는 색상 사이에 거의 보색 관계에 가까운 뒤바뀜 현상이 발생하고 있습니다. 이는 SVG 그라디언트 `grad-h` 내부의 색상 스톱 오프셋 순서가 HSL 표준 휠(Red -> Yellow -> Green -> Cyan -> Blue -> Magenta -> Red)의 완벽한 역방향으로 정의되어 있기 때문입니다. 이를 정방향 표준 HSL 컬러 스펙트럼으로 교정하여 화면 중앙선에 걸친 색상과 피킹되는 색조가 100% 동일하게 갱신되도록 해결합니다.

## 2. 핵심 기능
- **SVG Hue 스펙트럼 정방향화**:
  - `grad-h` 그라디언트의 각 스톱 오프셋 컬러를 HSL 색상환의 시계 방향(표준 순서)인 `Red(0%) -> Yellow(17%) -> Green(33%) -> Cyan(50%) -> Blue(67%) -> Magenta(83%) -> Red(100%)` 순으로 배치 교정합니다.
  - 이에 따라 슬라이더 트랜스폼 정방향 수식(`cy - v * trackLen`) 및 드래그 방향 정방향 수식(`v -= dv`)과 삼각 조합을 완벽히 이루어 시각 정보와 변수 데이터가 정확히 일치하게 됩니다.

## 3. 입력과 출력
- **입력**: 시작 시/조작 시의 `currentH` 변수값 및 드래그 변위.
- **출력**: 화면 가운데 라인에 표시되는 색상과 유저 피킹 HEX 색상 코드 간의 완벽한 일치.

## 4. 파일 구조
- [GameView.js](file:///e:/AI/DYE_MASTER/colorFinder/src/ui/GameView.js): Defs 내 `grad-h` 선형 그라디언트 정의 수정.

## 5. 핵심 모듈
- **Hue Gradient Alignment**: SVG Hue 색상 대역의 좌표계를 0도에서 360도로 순차 증가하는 표준 HSL 모델과 완벽 동기화.

## 6. 실행 흐름
1. H 슬라이더를 위아래로 드래그하면, `grad-h` 띠가 드래그 방향에 맞춰 매끄럽게 움직임.
2. 화면 중앙선에 놓이는 `v` 값 위치의 그라디언트 컬러(예: `v = 0.5` 일 때 Cyan)가 실제 변수 `currentH` 값(`180도` = Cyan)과 완벽하게 매칭되어 연산됨.

## 7. 에러 처리
- H=0 도와 H=360 도의 무한 루프 랩핑 경계면인 Red 영역이 접합선 노이즈 없이 스크롤되는지 검증.

## 8. 테스트 전략
- H 슬라이더를 드래그하여 화면 중앙선에 민트색/초록색 영역이 오도록 맞추었을 때, `#hex-display` 및 `guessBg`가 동일한 민트색/초록색 계열로 정확하게 피킹되는지 대조.
- 보라색/자주색 영역을 중앙선에 정렬시켰을 때 `#hex-display`가 보라색(예: `#8000ff`)을 정상 표출하는지 시각 검증.

## 9. 완료 기준
- H 슬라이더의 중앙 기준선 색상과 피킹 결과 색상이 완벽히 동치로 매치됨.
