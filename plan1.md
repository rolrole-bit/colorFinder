# DYE MASTER 배경 컬러 및 조작 피드백 지연 문제 해결 계획 (plan1.md)

## 1. 작업 목적
현재 슬라이더를 조작할 때마다 컬러 코드 텍스트(A, B, F 등)가 롤링되는 애니메이션(약 0.25초)과 함께 화면의 전체 배경, 헥스 박스, 중앙 기준선 색상 변경도 묶여 있어 화면의 피드백이 조작 속도를 즉각 따라가지 못하고 뒤늦게 부드럽게 쫓아오는 렌더링 딜레이 현상이 있습니다. 이를 텍스트 롤링 연출만 부드럽게 유지하고, 전체 시각적 배경과 컴포넌트의 컬러 동기화는 0 딜레이로 즉각 반응하도록 분리하여 플레이어에게 "착 달라붙는 조작감"을 제공합니다.

## 2. 핵심 기능
- **렌더링 즉시화**:
  - `updateColor()` 내에서 `guessBg`, `outerBg`, `submitBtn`, `hexDisplay`의 배경색과 텍스트 대비색(Contrast Color) 갱신 로직을 `requestAnimationFrame` 바깥으로 꺼내 `targetRGB` 목표값으로 즉각(`0ms`) 렌더링되게 분리합니다.
- **텍스트 롤링 애니메이션 유지**:
  - 숫자와 문자가 드르륵 굴러가는 텍스트 변환 연출(`hexDisplay.textContent` 업데이트)은 여전히 기존의 Ease-out(0.25초) 프레임 애니메이션 루프에 묶어 남겨둡니다.

## 3. 입력과 출력
- **입력**: 플레이어의 슬라이더 조작에 의해 발생한 실시간 H, S, L 값.
- **출력**: 지연 없이 0초 만에 화면 전역에 즉각 반영되는 조작 컬러 및 부드럽게 굴러가는 헥스 코드 텍스트 렌더링.

## 4. 파일 구조
- [GameView.js](file:///e:/AI/DYE_MASTER/colorFinder/src/ui/GameView.js): `updateColor` 내부의 비동기 렌더링 묶음 분리.

## 5. 핵심 모듈
- **Zero-Latency Rendering Dispatcher**: DOM 스타일 속성에 대해서는 보간 없이 즉시(target) 값을 꽂아 넣고, DOM 컨텐츠(텍스트) 속성만 보간(displayed) 값을 꽂아넣어 투트랙(Two-track) 렌더링을 구현합니다.

## 6. 실행 흐름
1. `currentH, S, L` 변화 감지 및 `updateColor()` 호출.
2. `immediateRgbStr`을 생성하여 DOM Style을 즉시 업데이트.
3. 텍스트 연출용 프레임 애니메이션 루프를 비동기 예약.
4. 애니메이션 프레임마다 `hexDisplay.textContent`만 업데이트하여 시각적 스무딩 달성.

## 7. 에러 처리
- `hexAnimFrame`이 매 `updateColor` 갱신마다 올바르게 취소 및 재생성되어 `textContent` 보간이 엉키지 않는지 보장.

## 8. 테스트 전략
- 슬라이더를 잡고 매우 빠른 속도로 위아래로 흔들었을 때 배경색상(`guessBg`)과 가운데 기준선이 고무줄처럼 뒤늦게 따라오지 않고 마우스 위치의 컬러를 즉각 반영하는지 손맛(촉각적) 검증.
- 그 와중에도 우측 상단의 컬러 코드 텍스트는 부드럽게 롤링 애니메이션을 유지하는지 대조.

## 9. 완료 기준
- 배경컬러 조작 피드백 딜레이가 완전히 제거되어 즉각적인 조작감을 제공함.
