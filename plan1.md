# DYE MASTER 최종 결과 화면 중앙 정렬 및 가로 폭 최적화 계획 (plan1.md)

## 1. 작업 목적
- **최종 결과 화면 가로 폭 최적화**: 현재 최종 결과 화면(스코어보드)의 레이아웃이 전체 화면 너비(`100vw`)로 넓게 퍼져 있어서 데스크톱 환경에서 가독성이 떨어지고 불안정한 느낌을 줍니다. 
- **시각적 가이드 정렬**: 콘텐츠의 최대 가로 폭을 최적의 크기(800px)로 제한하고 화면 중앙에 배치하여 집중도를 높이고 완성도 높은 매거진 스타일 레이아웃을 완성합니다.

## 2. 핵심 기능
- **중앙 정렬 래퍼 컨테이너 도입**: 최종 결과 화면 콘텐츠 전체를 감싸는 `.magazine-content` 클래스를 추가하여 가로 폭을 `max-width: 800px`, `width: 100%`로 제어합니다.
- **하단 버튼 레이아웃 정렬**: 하단에 고정된 "다시 하기" 및 "공유하기" 버튼 바의 최대 가로 폭도 콘텐츠 영역에 맞추어 `max-width: 800px`로 축소 및 정렬합니다.
- **반응형 웹 지원**: 모바일이나 태블릿 등 화면 폭이 800px보다 작은 기기에서는 자연스럽게 `100%` 너비로 채워지도록 반응형 분기를 보장합니다.

## 3. 입력과 출력
- **입력**: 게임 최종 스코어, 라운드별 점수, 플레이어/게임별 랭킹 리스트.
- **출력**: 모니터 해상도에 관계없이 중앙 `800px` 영역에 정돈되어 나타나는 수려한 스코어보드 UI.

## 4. 파일 구조
- [ScoreboardView.js](file:///e:/AI/DYE_MASTER/colorFinder/src/ui/ScoreboardView.js): 콘텐츠 영역 및 버튼 영역 HTML 구조에 `max-width` 및 정렬 클래스/스타일 적용.
- [index.css](file:///e:/AI/DYE_MASTER/colorFinder/src/index.css): `.magazine-content` 스타일 클래스 추가.

## 5. 핵심 모듈
- **Magazine Content Wrapper**: 콘텐츠의 폭을 제한하고 가로 정렬을 제어하는 레이아웃 모듈.

## 6. 실행 흐름
1. 게임 3라운드가 완료되면 최종 점수를 서버에 제출하고 `renderScoreBoardView`를 호출.
2. 스코어보드 뷰 렌더링 시, 뒷배경의 50:50 색상 분할 레이아웃은 화면 전체에 꽉 차게 렌더링됨.
3. 텍스트 및 랭킹 보드는 새롭게 정의된 `.magazine-content` 내부에 렌더링되어 중앙 `800px` 범위 내로 정렬됨.
4. 하단 버튼 바 또한 화면 가로 중앙 기준으로 최대 `800px` 너비에 맞게 배치되어 전체적인 정렬 라인이 일치함.

## 7. 에러 처리
- 화면 너비가 800px 미만일 때 패딩(`padding: 0 1.5rem`)을 주어 양옆 여백이 사라지고 텍스트가 벽에 붙는 현상을 방지함.

## 8. 테스트 전략
- 브라우저 개발자 도구의 디바이스 툴바를 활성화하여 FHD(1920x1080), QHD(2560x1440), 태블릿(768px), 모바일(375px) 해상도별 레이아웃 정렬 상태 검증.
- 하단 버튼의 양끝 라인이 플레이어 랭킹(좌) 및 게임별 랭킹(우) 보드의 양끝 라인과 정교하게 일치하는지 확인.

## 9. 완료 기준
- 데스크톱 해상도에서 최종 결과 화면의 텍스트와 랭킹 목록이 최대 가로 800px 크기로 제한되어 화면 정중앙에 수려하게 노출됨.
- 다시 하기 / 공유하기 버튼 바가 상단 콘텐츠 카드 폭(800px)과 오차 없이 정렬됨.
