# DYE MASTER 가운데 라인 컬러 피킹 불일치 해결 계획 (plan1.md)

## 1. 작업 목적
현재 HSL 슬라이더 조작 시, 화면 가운데 배치된 흰색 기준선(Center Line)에 맺히는 시각적인 컬러 띠의 위치와 실제 피킹되는 논리적 수치(H, S, L 값)가 서로 불일치하여 거꾸로 반영되는 심각한 오류가 발생하고 있습니다. 이를 해결하여 기준선에 걸치는 색상이 플레이어의 최종 선택 색상으로 한 치의 오차도 없이 명확하게 반영되도록 보정합니다.

## 2. 핵심 기능
- **수직 매핑 방향성 일치**: `TapeSlider.updateTransform()`의 Y좌표 계산식을 기존의 역방향 `cy - (1 - v) * trackLen`에서 정방향 `cy - v * trackLen`으로 교정합니다.
- **S, L 논리적 대칭 변환 교정**: 
  - S와 L 그라디언트 띠는 가운데(50% 지점)가 가장 어두운 대칭 구조이므로, 물리적인 슬라이더 값(`0~200`)을 논리적인 값(`0~100`)으로 바꿀 때의 반사 수식을 `val > 100 ? val - 100 : 100 - val`로 수정하여 시각적인 위치와 논리 수치가 일대일 매핑되도록 보정합니다.
- **초기 렌더링 값 역산 보정**: 게임 시작 시 타겟 색상으로부터 랜덤하게 분산된 초기 채도/명도 값이 슬라이더의 물리적 초기 위치에 완벽하게 정렬되도록 `100 - currentS/L` 형태로 역산하여 지정합니다.

## 3. 입력과 출력
- **입력**: 시작 시 분산된 `currentS/L` 색상 값, 조작 시 드래그 수치.
- **출력**: 화면 가운데 기준선에 노출되는 그라디언트 색상과 일치하는 사용자 최종 선택 색상(HEX 코드 및 피킹 결과).

## 4. 파일 구조
- [GameView.js](file:///e:/AI/DYE_MASTER/colorFinder/src/ui/GameView.js): 슬라이더 Y 트랜스폼 계산식 수정, 슬라이더 생성자 매개변수 역산 적용, 대칭 변환 공식 업데이트, 셔플 로직 변환 보정.

## 5. 핵심 모듈
- **Transform Synchronizer**: 슬라이더 값과 물리적으로 화면에 표시되는 테이프 Y 좌표를 정방향으로 매핑.
- **Symmetry Normalizer**: `val > 100 ? val - 100 : 100 - val` 공식을 통해 0~200 물리 좌표를 0~100~0 논리 좌표로 정확하게 매칭.

## 6. 실행 흐름
1. 게임 시작 시 `currentS/L`에 대해 `100 - currentS/L`을 적용하여 `satSlider/lightSlider` 인스턴스 생성.
2. 사용자가 드래그하면 `updateTransform()`이 `cy - v * trackLen` 식으로 테이프 위치를 정밀 제어.
3. 드래그 중인 물리적 누적 값 `val`은 대칭 정규화 공식을 거쳐 완벽한 HSL 좌표로 치환되어 `#hex-display` 및 `guessBg`에 반영.

## 7. 에러 처리
- 인트로 셔플 믹싱이 완전히 종료되는 순간 물리적인 초기 위치(`100 - startS/L`)로 깔끔하게 원복되도록 예외 처리 보강.

## 8. 테스트 전략
- L 슬라이더의 가운데 기준선에 어두운 검정색이 걸치도록 드래그했을 때, 사용자 컬러가 동일하게 어두워지고 헥스코드가 어두운 색상값으로 정확히 바뀌는지 확인.
- 반대로 가운데 기준선에 밝은 백색이 걸칠 때 사용자 컬러와 헥스코드가 백색(밝은 톤)으로 정확하게 대응되는지 역방향으로 재검증.

## 9. 완료 기준
- 화면 가운데 흰색 가로선에 위치한 색상과 최종 피킹되는 유저 컬러가 100% 동일하게 갱신됨.
