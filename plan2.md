# PLAN 2: 패션 매거진 스타일 결과 화면 개편 (UI/UX)

## UI 필요 여부
- **필요함**: 최종 결과 화면(ScoreboardView) 및 라운드 결과 화면(ResultView)의 전면적 스타일링 개편.

## 사용자 흐름
1. 게임 완료 후 최종 점수 획득.
2. 스코어보드 화면 진입.
3. 거대한 점수 텍스트와 세밀한 실선으로 표현된 라운드 성적표 확인.
4. 아래로 스크롤하여 세련되게 정돈된 플레이어 랭킹과 게임별 랭킹 확인.
5. 하단의 버튼(`다시 하기`, `공유하기`)을 통해 흐름 분기. (버튼이 글자를 가리지 않음)

## 화면 및 명령 흐름
- **기존 문제**:
  - 버튼 컨테이너가 `fixed`로 고정되어 랭킹의 아래 내용을 덮어버림.
  - 카드 형태의 둥근 모서리와 겹치는 테두리가 매거진 컨셉에 어긋남.
- **개선안**:
  - 스코어 화면의 최하단에 버튼 영역을 두고, 스크롤 가능한 본문 컨테이너에 하단 패딩(`padding-bottom: 8rem`)을 제공하여 랭킹 목록이 끝까지 올라와 가려지지 않게 함.
  - 혹은 버튼 영역에만 반투명 그라데이션 차단막(`backdrop-filter: blur(8px)`)을 씌운 하단 고정 바를 구성하여, 리스트가 그 아래로 지나가더라도 겹친 텍스트가 번지지 않게 함. (매거진 하단 푸터 느낌의 바 구성)

## core 모듈과의 연동 방식
- `GameState.js` 의 상태값을 그대로 사용하며 UI 변경만 유도함.

## 상태 처리
- 없음 (기존 세션 데이터 및 랭킹 데이터 유지).

## 에러 표시
- 대비색이 일치하지 않아 글자가 안 보이는 현상이 없도록 JS `leftContrast`/`rightContrast` 색상을 랭킹의 세부 요소와 보더라인에 주입.

## 검증 방법
- 웹브라우저 창 크기를 조절하여 반응형(Mobile ~ Desktop) 상태에서도 글자가 겹치지 않고 그리드가 일그러지지 않는지 테스트.
