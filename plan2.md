# DYE MASTER 인트로 연출 시퀀스 타겟 컬러 기반 셔플 화면/동작 흐름 계획 (plan2.md)

## 1. UI 필요 여부
- 구조적 UI 레이아웃의 변경은 없습니다. 초기 데이터 세팅 위상 구조의 변경입니다.

## 2. 사용자 흐름
1. **타겟 색상 기억 (3초)**: 게임이 시작되고 사용자가 화면 가득 채워진 정답 색상을 인지합니다.
2. **추측 화면 진입 시점**: 화면이 분할(Split) 레이아웃으로 변경되지만, 좌/우측의 컬러는 똑같은 정답 색상으로 렌더링되어 있습니다. (이전에 발생하던 갑작스러운 1프레임 튀는 위상 불일치가 완벽하게 사라집니다).
3. **셔플 애니메이션 발동 (1.5초)**:
   - "정답 위치에 멈춰 있던 슬라이더들이 서서히 이동하며 사용자가 맞춰야 할 위치를 임의로 흩트려놓는(Shuffle)" 내러티브 연출이 전개됩니다.
   - H, S, L 슬라이더가 0.2초 간격으로 스르륵 움직이며 화면의 우측(사용자 컬러) 색상을 부드럽게 섞기 시작합니다.
4. **힌트 종료 및 조작 시작**: 슬라이더가 단 한 방향으로 이동을 마친 뒤 안정적으로 멈추며, 이때부터 사용자가 조작하여 원래의 정답으로 되돌아가는 여정을 시작합니다.

## 3. 핵심 모듈과의 연동 방식
- `GameView.js` 내의 상태 초기화 로직 분리: `initHsl`과 시작점 `current` 값을 `tH, tS, tL`로 고정.
- 목적지 값을 분리 선언: `endH, endS, endL`.
- `updateColor()`와 연계된 `doShuffle` 내부 로직이 `end` 값을 향해 정방향 추적(Interpolation)을 수행하도록 수식 전면 수정.

## 4. 상태 처리 및 연동
- 애니메이션 도중 생성되는 `hVal`, `sVal`, `lVal` 보간값을 바탕으로 테이프 슬라이더의 `value`를 실시간 계산하여 대입. S, L은 100 기준 대칭 변환(`100 - x`) 공식을 적용하여 시각적 위상 동기화를 보장함.

## 5. 검증 방법
- 추측 화면 진입 시 "뿅" 하고 컬러가 튀는 현상이 1프레임도 허용되지 않고 즉각 정답 화면부터 부드럽게 셔플되는지 캡처 및 눈으로 확인.

## 6. 확장 가능성
- 추후 '하드모드' 도입 시, 섞어버리는 거리 오프셋(`hueOffset` 등)의 상한선을 높이거나, 셔플 시간을 0.5초로 극도로 짧게 만들어 긴장감을 부여하는 등, 셔플 파라미터 튜닝이 매우 직관적인 형태(출발지 -> 목적지)로 구조화됩니다.
