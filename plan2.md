# DYE MASTER 인트로 연출 단방향 천천히 이동 화면/동작 흐름 계획 (plan2.md)

## 1. UI 필요 여부
- UI 레이아웃 변경은 없고, 기존 애니메이션의 연출 보간 수식만 변경합니다.

## 2. 사용자 흐름
1. **타겟 색상 노출**: 게임 시작 후 3초간 타겟 컬러를 인지합니다.
2. **추측 화면 전환**: 3개의 조작 슬라이더가 나타납니다.
3. **단방향 힌트 애니메이션 (1.5초)**:
   - 3개의 슬라이더가 미리 정해진 시작점(타겟에서 약간 어긋난 위치)보다 조금 더 위쪽(혹은 아래쪽)으로 당겨진 상태로 대기하고 있습니다.
   - 시간이 흐르며 차례차례(H -> S -> L 순서) 부드럽게 미끄러지듯 이동하여 본래의 시작점(스타트 오프셋 지점)에 안착합니다.
   - 왔다갔다(왕복)하지 않으므로 어지럽거나 화면이 요동치는 느낌이 완전히 배제됩니다.

## 3. 핵심 모듈과의 연동 방식
- `GameView.js` 내의 `doShuffle` 함수의 보간 기법을 `Math.sin`(왕복 파형)에서 `Cubic Ease-Out`(단방향 감속 도착 파형)으로 대체.
- 계산된 단방향 비율(0~1)을 이용해 떨어진 거리(`-40`, `-25` 등)를 `0`으로 서서히 줄여나가는 공식을 적용.

## 4. 상태 처리 및 연동
- 애니메이션 진행 값에 따라 `startH, startS, startL`에 오프셋 보정치(`-40 * (1 - ease)`)를 실시간 가산하여 슬라이더 물리 좌표 갱신.

## 5. 검증 방법
- 1라운드 시작 시, 색상이 확 바뀌지 않고 처음 진입한 그 색상 톤에서 아주 약간의 미끄러짐만 발생한 후 정지하는지 체감 확인.
- 속도가 1.5초(1500ms)로 길어져서 이전보다 훨씬 여유롭고 차분하게 안내되는지 검증.

## 6. 확장 가능성
- 향후 슬라이더 미끄러짐의 진폭(현재 -40, -25)을 늘리거나 줄이고 싶을 경우, 변수만 조정하여 손쉽게 진폭을 확장/축소 제어할 수 있습니다.
