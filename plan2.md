# DYE MASTER 추측(GUESS) 화면 컬러 시인성 개선 계획 (2)

## UI 필요 여부
- **필요함**: 기존 추측(GUESS) 단계의 전면 스크린 SVG 슬라이더를 중앙 50% 수준으로 모으고 좌우 여백을 온전히 활용하기 위해 CSS 및 HTML/SVG 렌더링 변경이 필수적임.

## 사용자 흐름
1. 게임 시작 후 타겟 색상을 3초간 기억.
2. 추측(GUESS) 페이즈로 전환되면 화면 중앙에만 세련되게 모인 3개의 세로형 HSL 슬라이더가 렌더링됨.
3. 좌우 여백에는 사용자가 현재 섞고 있는 최종 색상(Solid Color)이 흐림 없이 보여짐.
4. 사용자가 중앙의 슬라이더들을 조작하면 좌우 영역의 색상이 부드럽게 연동되어 바뀜.
5. 원하는 색상에 도달하면 하단의 'DONE'을 눌러 제출.

## 화면 또는 명령 흐름
- **기존 구조**:
  - `svg-sliders-container`가 `width: 100%`, `height: 100%`를 통째로 덮음.
  - 블러 오버레이 `backdrop-filter: blur(80px)`를 가진 div도 `inset: 0`으로 화면 전체를 가림.
- **새로운 구조**:
  - 블러 오버레이와 SVG 슬라이더 컨테이너를 하나로 묶어 **중앙 50%** 영역(PC 기준)에만 가둡니다.
  - 구체적으로 `svg-sliders-container`와 `blur-overlay`의 스타일을 다음과 같이 수정합니다.
    - `left: 25%`, `width: 50%` (혹은 CSS 클래스로 정의하여 모바일에선 `left: 15%`, `width: 70%`로 유동적 할당)
  - SVG 내부 좌표계(`rect x="0%"`, `rect x="33.3%"`, `rect x="66.6%"`)는 슬라이더 박스 내부 비율이므로 그대로 유지해도 컨테이너 너비만 50%로 줄여주면 자동으로 중앙에 모이게 됩니다.

## core 모듈과의 연동 방식
- 사용자 조작에 따른 H, S, L 값 변화 및 최종 RGB 변환 로직(`hslToRgb`, `setUserColor`)은 기존 `GameState.js` 및 `GameView.js` 내 구현된 코드를 그대로 유지하여 안전성 확보.

## 상태 처리
- 사용자가 조작 중인 `currentH`, `currentS`, `currentL` 상태는 실시간으로 반영되고, `guess-bg` 배경색을 실시간으로 업데이트.
- 슬라이더 드래그 시 좌표가 좁아진 화면 비율에 맞춰 정상적으로 스케일링되도록 `svgContainer.clientHeight` 등 `TapeSlider` 클래스 내 계산 공식에 문제없는지 재점검.

## 에러 표시
- 해상도 변화 시 슬라이더 중심선(`center-line`) 및 가이드가 어긋나지 않도록 SVG의 `y1="50%"` 및 `y2="50%"` 기반 퍼센트 레이아웃 고수.

## 검증 방법
- 웹브라우저 창 가로 길이를 수시로 늘리고 줄이며 반응형(Responsive) 처리가 자연스러운지 확인.
- F12 개발자 도구를 사용하여 모바일 뷰(iPhone SE 375px ~ iPad)에서 슬라이더를 부드럽게 끌었을 때, 오차 없이 정확히 드래그한 포인터를 잘 따라오는지 및 양옆 배경색이 눈에 선명하게 드러나는지 수동 검증.

## 확장 가능성
- 차후 테마 변경이나 다크 모드 등을 지원할 경우, 중앙 슬라이더 박스 배경에 미세한 테두리(`border: 1px solid rgba(255,255,255,0.1)`)를 넣거나 섀도우를 주는 디자인 시스템 확장이 매우 용이함.
