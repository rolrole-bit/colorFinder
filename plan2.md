# DYE MASTER 인트로 힌트 연출 개선 화면/동작 흐름 계획 (plan2.md)

## 1. UI 필요 여부
- UI 레이아웃의 구조 변경은 없으며, 애니메이션 연출 렌더링 방식의 변경입니다.

## 2. 사용자 흐름
1. **타겟 색상 노출**: 게임이 시작되고 3초간 타겟 색상을 외웁니다.
2. **추측 화면 진입**: 화면이 추측용 슬라이더 화면으로 전환됩니다.
3. **조작 힌트 애니메이션 (1초)**:
   - 왼쪽의 H 슬라이더가 먼저 위아래로 살짝 들썩입니다. (0 ~ 0.7초)
   - 이어서 S 슬라이더가 약간의 딜레이를 두고 살짝 들썩입니다. (0.15 ~ 0.85초)
   - 마지막으로 L 슬라이더가 딜레이를 두고 살짝 들썩입니다. (0.3 ~ 1.0초)
4. **힌트 종료**: 세 슬라이더가 모두 본래 시작 오프셋 위치로 수렴하며 안정을 찾습니다. 전체적인 배경 색상은 1초 동안 어지럽게 뒤섞이지 않고 잔잔한 물결처럼만 미세하게 변동됩니다.
5. **터치 개입**: 만약 연출 도중에 사용자가 슬라이더 영역에 손을 대면, 연출이 그 즉시 파괴되고 사용자의 수동 조작으로 매끄럽게 연결됩니다.

## 3. 핵심 모듈과의 연동 방식
- `GameView.js` 내의 `doShuffle` 함수 내부에서 수학적 보간식 교체.
- `calcWave(progress, start, end)` 독립 헬퍼 함수를 통해 슬라이더별 위상 및 진폭 제어.

## 4. 상태 처리 및 연동
- 애니메이션 진행 상태값 `progress` 변수를 분할 재가공하여 슬라이더 각각에 대한 고유한 움직임 배수(`easeH`, `easeS`, `easeL`) 도출.

## 5. 검증 방법
- 1라운드 진입 시 슬라이더들이 각각 순차적으로 "나 여기 있어"라고 알려주는 듯한 개별적인 움직임을 시각적으로 보여주는지 느낌 검증.
- 화면 색상이 눈 아프게 깜빡이거나 크게 요동치는지 여부 검토. (작은 진폭이 적용되었으므로 요동치지 않아야 함)

## 6. 확장 가능성
- 차후 튜토리얼 라운드와 본게임 라운드에서 힌트 연출의 길이 및 강도를 다르게 주고 싶을 경우, `doShuffle`에 난이도별 파라미터를 넘겨받도록 확장할 수 있습니다.
