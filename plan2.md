# DYE MASTER 슬라이더 무한 대칭 루프 화면/동작 흐름 계획 (plan2.md)

## 1. UI 필요 여부
- UI 그래픽의 대규모 보정이 필요합니다.
- 기존에 `y="-3000"` 등 단색 사각형으로 막혀있던 S, L 테이프 슬라이더의 SVG 구성을 H 슬라이더와 동일하게 **정방향 1500px 크기의 rect 3장(상/중/하)을 이어 붙인 3단 테이프 구조**로 변경합니다.
- 이에 맞추어 `grad-s`와 `grad-l` 그라디언트를 대칭(정방향+역방향이 결합된 0~100~0)으로 다시 구성하여 접합선이 육안으로 드러나지 않는 끊김 없는 무한 스페이스를 창조합니다.

## 2. 사용자 흐름
1. **슬라이더 터치 및 이동**: 사용자가 채도(S) 또는 명도(L) 슬라이더를 잡고 드래그합니다.
2. **경계선 도달 (0% 또는 100%)**: 
   - 기존에는 0이나 100에서 썸과 트랙이 딱 멈춰 서서 더 이상 이동하지 않았으나, 이제는 계속 드래그를 진행할 수 있습니다.
   - 드래그를 계속 진행하면 슬라이더의 물리적 이동 방향을 따라 배경 그라디언트가 끊김 없이 끝없이 흘러갑니다.
   - 이때 채도 값은 예컨대 `100% -> 90% -> 80% ...`로 부드럽게 감소(대칭 왕복)되어 색조가 부드럽게 변화합니다.

## 3. 핵심 모듈과의 연동 방식
- `TapeSlider` 내부의 변위 누적(`v`) 및 업데이트 공식을 360 조건 분기 없이 전체 공통 무한 순환으로 통일.
- `updateColor()` 함수에서 `grad-s`와 `grad-l` 내부의 동적 스톱 컬러 변경 시, 대칭점들을 모두 갱신해주도록 변경:
  - `sat-stop-0` (0%), `sat-stop-1` (50%), `sat-stop-2` (100%)
  - `light-stop-mid1` (25%), `light-stop-mid2` (75%)

## 4. 상태 처리 및 연동
- 슬라이더 값 범위:
  - Hue: `0 ~ 360` (동일)
  - Saturation: `0 ~ 200` (논리적 `0~100` 왕복 변환)
  - Lightness: `0 ~ 200` (논리적 `0~100` 왕복 변환)
- 셔플 믹싱 공식 역시 슬라이더 인스턴스의 변경된 물리 범위(`0~200`)로 작동되도록 보간 연산 조율.

## 5. 검증 방법
- 슬라이더들을 매우 빠른 속도로 아래 또는 위로 끝없이 올리고 내릴 때, 뚝 끊기거나 반짝이는 이음새 에러가 전혀 생기지 않는지 확인.
- 값이 증가하다가 정점에 이르면 다시 부드럽게 복귀 및 감소하는 핑퐁 왕복이 컬러 코드 디스플레이 및 실제 색상 갱신과 완벽히 부착하는지 체크.

## 6. 확장 가능성
- 차후 모바일의 협소한 디바이스 영역에서 슬라이더를 넓게 드래그하지 못하는 한계를 보완하기 위해, 드래그 감도(Sensitivity)를 옵션으로 두어 한 번의 쓸어넘김으로 몇 바퀴든 부드럽게 루프 돌릴 수 있도록 설계 변경을 용이하게 합니다.
