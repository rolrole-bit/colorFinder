# DYE MASTER 배경 컬러 즉각 피드백 화면/동작 흐름 계획 (plan2.md)

## 1. UI 필요 여부
- 추가적인 UI의 생성이나 삭제는 없으며, 시각적인 컬러 갱신을 이원화(Immediate Style Update vs Animated Text Update)하는 성능적 렌더링 로직 개편입니다.

## 2. 사용자 흐름
1. **슬라이더 조작**: 플레이어가 슬라이더를 잡고 매우 빠른 속도로 스와이프합니다.
2. **컬러 동기화(즉각 반영)**: 게임 화면 절반의 배경색(`guessBg`), 가장자리 배경색(`outerBg`), 중앙 기준선, 버튼 색상 등 모든 그래픽 색상이 슬라이더 움직임과 오차 없이 딱딱 맞물려 **0.001초의 딜레이도 없이 즉시(0ms)** 변합니다. 
3. **텍스트 연출**: 우측 상단의 컬러 코드 숫자판은 기존처럼 드르르륵 굴러가며 사용자에게 고급스러운 애니메이션 감각을 그대로 선사합니다. 조작감은 민첩하게 유지하면서도 눈에 보이는 퀄리티는 유지되는 완벽한 밸런스가 달성됩니다.

## 3. 핵심 모듈과의 연동 방식
- `GameView.js` 내의 `updateColor` 함수 내부 로직을 분리.
- `DOM.style` 접근은 `targetRGB` 변수를 이용해 동기(Synchronous) 블록에서 처리.
- `hexDisplay.textContent` 접근은 보간된 `currentDisplayedRGB` 변수를 이용해 비동기(Asynchronous / `requestAnimationFrame`) 블록에서 처리.

## 4. 상태 처리 및 연동
- 보간 변수 `currentDisplayedRGB`는 여전히 애니메이션 프레임을 통해 타겟을 쫓아가지만 그 적용처를 `textContent` 텍스트 하나로 제한.

## 5. 검증 방법
- 마우스를 마구 흔들었을 때 배경색과 슬라이더 컬러가 고무줄처럼 나중에 따라오는 느낌("꿀렁꿀렁")이 사라졌는지 체감 검토.

## 6. 확장 가능성
- 차후 숫자 롤링 애니메이션의 재생 시간을 현재의 250ms가 아닌 옵션으로 조작하게 되더라도, 배경 렌더링과는 완전히 분리되었으므로 부작용(사이드 이펙트) 없이 안전하게 수정이 가능해집니다.
