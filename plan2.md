# DYE MASTER 조작 가이드 화살표 화면/동작 흐름 계획 (plan2.md)

## 1. UI 필요 여부
- UI 요소를 새롭게 추가합니다. 
- `GameView`의 HSL 슬라이더 위에 튜토리얼 레이어(`div#tutorial-overlay`)를 중첩(z-index: 15) 배치하여 사용자가 인지하도록 돕습니다.
- 화살표 형태는 SVG 파일로 직접 하드코딩 렌더링하며, CSS Absolute 포지션을 사용하여 슬라이더의 중앙 수직 라인(16.66%, 50%, 83.33%)에 화합하도록 배치합니다.

## 2. 사용자 흐름
1. **1라운드 진입**: 본게임 1라운드(`ROUND 1`)에 들어오면 슬라이더의 동작 힌트(사인파 무빙)가 부드럽게 재생되면서 그 위에 **3개의 수직 양방향 점선 화살표**가 투명도 0.85로 뚜렷하게 노출됩니다.
2. **슬라이더 터치/드래그 시작**: 
   - 플레이어가 임의의 슬라이더 터치 영역을 터치하거나 조작을 개시(PointerDown)합니다.
   - 즉시 튜토리얼 레이어가 트리거되어 `opacity = 0`으로 수렴하며 스르륵 사라집니다. (0.8초 트랜지션)
   - 소멸 과정이 종료(transitionend)되면 DOM 트리에서 완전 제거됩니다.
3. **이후 라운드 (ROUND 2, 3)**:
   - 한 번 조작법을 익혔으므로 ROUND 2, 3에서는 해당 안내 화살표가 렌더링되지 않아 불필요하게 시야를 가리지 않습니다.

## 3. 핵심 모듈과의 연동 방식
- `GameView.js` 내의 `transitionToGuess` 내부에서 `state.currentRound` 상태 변수를 확인하여 렌더링 유무 결정.
- `TapeSlider` 인스턴스의 `onDown` 메서드가 직접 사용자의 입력 감지 인터럽트를 트리거하여 `fadeTutorial()` 함수를 안전하게 호출.

## 4. 상태 처리 및 연동
- `isFirstRound`: `state.currentRound === 1`로 판단하여 튜토리얼 활성화 여부 제어.
- `tutorial-overlay`: 가이드 화살표를 감싸고 있는 최상위 돔 래퍼.

## 5. 검증 방법
- ROUND 1 추측 화면 기동 시 3개 화살표가 올바른 위치에 오프셋 정렬되는지 시각적 대조.
- 터치 즉시 화면에서 투명도가 부드럽게 감속하며 자연스럽게 지워지는지 확인.
- 지워진 이후 DOM에 잔여 노드가 남아 이벤트 리스너 리소스를 먹지 않는지 개발자 도구로 검증.

## 6. 확장 가능성
- 모바일 가로모드나 데스크톱 전체화면 등 다양한 기기 폭에 맞춰 3개 슬라이더가 33.34% 너비를 나누어 갖는 구조이므로, 화살표 X좌표 역시 CSS 백분율(`left: 16.66%`, `50%`, `83.33%`)로 설계해 반응형 환경에 완전 대응되도록 합니다.
