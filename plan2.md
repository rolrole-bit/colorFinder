# DYE MASTER 가운데 라인 컬러 피킹 불일치 화면/동작 흐름 계획 (plan2.md)

## 1. UI 필요 여부
- 추가적인 UI 노드는 없으나, 기존 `TapeSlider`에 연결된 테이프 SVG 그래픽의 Y좌표 연산(`updateTransform`)의 정합성을 완전히 재설계합니다.
- 화면 중앙의 가로형 흰색 라인(`#center-line`)을 기준으로 삼아 정확히 피킹이 이루어지게 수정합니다.

## 2. 사용자 흐름
1. **슬라이더 조작**: 플레이어가 명도(L) 슬라이더를 아래로 끌어내립니다.
2. **기준선 시각적 피킹**: 테이프가 위로 올라가면서 화면 기준선에 어두운 검정색 그라디언트 영역이 도달합니다.
3. **색상 및 헥스코드 동기화**: 즉시 `#hex-display` 및 `guessBg` 영역의 색상이 동일한 톤의 어두운 색상으로 갱신되어, 플레이어가 "내가 기준선에 맞추고 있는 색상이 지금 선택되고 있구나"를 온전히 인지할 수 있습니다.

## 3. 핵심 모듈과의 연동 방식
- `TapeSlider`의 `updateTransform` 계산식을 `cy - v * trackLen`으로 교체.
- 슬라이더의 `onChange` 내 아웃풋 매핑 수식을 `val > 100 ? val - 100 : 100 - val` 형태로 뒤집어, 대칭형 그라디언트 띠 스톱 컬러 분포와 매칭.
- 게임 시작 시 `currentS/L`로부터 슬라이더의 물리적 시작 좌표를 역산(`100 - currentS/L`)하여 배치.

## 4. 상태 처리 및 연동
- `satSlider.value` / `lightSlider.value`의 초기화 시점 값을 `100 - currentS/L`로 적용.
- `onChange` 변경 시 `currentS`와 `currentL`의 상태값을 새로운 대칭 공식을 통해 가공 저장.
- 인트로 셔플 종료 시에도 물리 좌표 복구용 변수로 `100 - startS/L` 적용.

## 5. 검증 방법
- 1라운드 시작 시, 분산된 슬라이더들의 화면상 위치와 실제 갱신된 `guessBg` 컬러 사이에 인지적 어긋남이 없는지 확인.
- L 슬라이더의 가운데 기준선에 검은색이 닿았을 때 `#hex-display`가 이에 대응하는 저명도 값(예: `#121212`)을 정확히 출력하는지 확인.
- S 슬라이더 역시 기준선에 Gray 영역이 닿았을 때 채도가 완전히 빠지는지 확인.

## 6. 확장 가능성
- 기준선이 화면 정중앙(`cy`)이 아닌 임의의 Y 좌표로 디자이너의 요구에 의해 변동되더라도, `cy` 값만 바꾸어주면 Y좌표 매핑이 완벽하게 일체형으로 호환되도록 구성합니다.
