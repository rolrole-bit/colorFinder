# Plan 2: DYE MASTER 런처 UI 디자인 및 사용자 경험 최적화 계획

## 1. UI 필요 여부
- **필요함**: `launcher.hta`는 DYE MASTER의 서버 실행 및 빌드를 돕는 전용 그래픽 사용자 인터페이스(GUI) 프로그램이다.
- **개선 사유**: 기존 UI는 윈도우 창 크기가 과도하게 커서 미관상 좋지 않고, HTA 기본 렌더링 엔진의 한계로 flexbox 스타일링 및 한글 폰트 인코딩이 깨져 가독성이 저하되었다.
- **향후 확장 방식**: 런처 내에 서버 종료(Kill Process) 기능, 포트 사용자 지정 옵션 등을 추가할 수 있도록 확장성을 고려하여 레이아웃을 설계한다.

## 2. 사용자 흐름
1. 사용자가 납품 폴더에서 `launcher.hta`를 더블클릭하여 실행한다.
2. 런처 창이 모니터 화면 중앙에 760x450px 크기로 아담하고 깔끔하게 팝업된다.
3. 런처 내의 텍스트와 타이틀이 깨짐 없이 한글로 선명하게 출력된다.
4. 사용자가 카드를 클릭하면 해당하는 Node.js 서버 프로세스가 성공적으로 생성된다.

## 3. 화면 또는 명령 흐름
- **창 초기화**:
  - HTA 파일 로드 시 `<head>` 영역에서 최신 Edge 렌더링 엔진을 사용하도록 정의.
  - HTML 내 인라인 자바스크립트를 활용해 `window.resizeTo(760, 460)`과 중앙 배치 계산 스크립트를 즉각 수행.
- **레이아웃 구조**:
  - `DYE MASTER` (대타이틀)
  - `SERVER LAUNCHER` (소타이틀)
  - 카드 섹션 (개발 서버, 배포 서버, 빌드 도구) - 가로 flex 정렬
  - 상태 메시지 영역 (클릭 결과 및 URL 링크 노출)

## 4. Core 모듈과의 연동 방식
- `launcher.hta`는 프론트엔드 HTML 환경이며, 사용자가 클릭 시 ActiveXObject(`WScript.Shell`)를 활용하여 로컬 Command Line 도구를 실행한다.
- `cmd /c`를 통해 `node server/index.js` 혹은 `node tools/build.js`를 백그라운드 또는 포그라운드로 실행한다.

## 5. 상태 처리
- **기본 상태**: "카드를 클릭하여 서버를 시작하세요" (회색 텍스트)
- **서버 구동 성공**: "✅ [개발/배포] 서버 시작됨! http://localhost:PORT" (초록색 텍스트 및 링크 활성화)
- **오류 발생**: "⚠ dist/ 폴더가 없습니다." 또는 "❌ 서버 시작 실패" (빨간색 텍스트)

## 6. 에러 표시
- HTA 내 상태 메시지 영역에 직관적인 텍스트 컬러(Red)와 상세한 가이드 메시지(HTML 태그 포함)를 노출하여 사용자가 에러 상황을 명확하게 파악할 수 있도록 유도한다.

## 7. 검증 방법
- `launcher.hta` 파일을 직접 실행하여 창 크기가 의도한 대로 줄어들었는지 검증한다.
- 메인 타이틀, 서브 타이틀 및 카드 텍스트의 한글이 깨짐 없이 미려하게 출력되는지 확인한다.
- 실제로 "개발 서버", "배포 서버" 버튼을 클릭하여 Node.js 콘솔 창이 정상적으로 팝업되는지 확인한다.

## 8. 확장 가능성
- 차후 런처에서 포트를 직접 변경하는 입력창(input)을 넣거나 현재 실행 중인 서버 프로세스의 상태를 주기적으로 감지하여 온/오프 상태를 표시하도록 기능을 확장할 수 있다.
