# UI 필요 여부
- 직접적인 신규 UI 추가는 없으나, 기존의 모든 UI(결과 화면, 스코어보드, 게임 화면 등)의 텍스트 가시성이 세련되게 개선됩니다.

# 사용자 흐름
1. 사용자가 게임을 진행하거나 결과 화면에 도달합니다.
2. 다양한 배경색(타겟 컬러, 유저 컬러 등)이 50:50 분할 배경이나 블러 백그라운드로 깔립니다.
3. 그 위에 표시되는 텍스트들이 배경색의 HSL 보색이면서 가독성이 확보된 명도로 렌더링되어, 시각적 피로감 없이 정보가 명확히 보입니다.

# 화면 또는 명령 흐름
- UI 렌더링 시점에 인라인 스타일 `color: getHSLContrastColor(...)`를 통해 색상 자동 주입

# core 모듈과의 연동 방식
- `ColorUtils.js`에 계산 함수를 탑재하고 이를 UI 모듈(`ResultView.js`, `ScoreboardView.js`, `GameView.js`)에서 임포트하여 실행

# 상태 처리
- 상태 변경 시(예: 라운드 종료, 최종 스코어 페이지 진입 등) 배경색이 변경되면 그에 부합하는 대비색이 동적으로 다시 계산되어 즉시 적용됩니다.

# 에러 표시
- 유효하지 않은 입력 데이터 발생 시 브라우저 콘솔에 경고를 출력하고, `#ffffff` 또는 `#000000`을 fallback으로 적용하여 글자가 보이지 않는 화면 먹통 현상을 방지합니다.

# 검증 방법
- 중간 명도 배경(예: 핑크 `#e08090`, 연두 `#90e080`)의 뷰를 로드하여 텍스트가 묻히지 않고 확실한 보색 및 높은/낮은 명도로 밀려나 렌더링되는지 브라우저에서 직접 시각적으로 확인합니다.

# 확장 가능성
- 차후 테마 설정이나 텍스트 컬러 프리셋이 추가될 경우, 이 대비 공식을 기본 엔진으로 활용하여 대비 감도를 사용자 옵션에 따라 다르게 줄 수 있도록 파라미터화(`threshold`, `pushLightness`)할 수 있는 구조로 작성합니다.
