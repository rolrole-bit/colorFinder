# S-Curve 기반 네오 브루탈리즘 슬라이더 구현 계획 (plan2.md)

현재의 일자형 수직 슬라이더를 첨부해주신 레퍼런스(Music OS)와 같은 **S자 곡선(S-Curve) 트랙 슬라이더**로 완전히 개편하는 계획입니다. 

## UI 필요 여부
- UI 개편 작업입니다. 기존의 직선 슬라이더 UI를 커스텀 SVG 곡선 렌더링 방식으로 대체합니다.

## 사용자 흐름
1. 사용자가 게임 화면에 진입하면, 화면 중앙 혹은 좌측에 유려한 S자 곡선 3개가 평행하게 렌더링됩니다.
2. 각 곡선 위에는 커다란 원형 조작부(Thumb)가 위치합니다.
3. 사용자가 조작부를 드래그하면, 조작부는 반드시 곡선을 따라서만 움직이며(스냅), 곡선의 수직 높이(Y) 비율에 따라 H, S, L 값이 변동됩니다.

## 화면 또는 명령 흐름
- GameView.js 렌더링 시 `<svg>` 및 `<path>` 생성 -> `PathSlider.js` (신규 혹은 수정된 CustomSlider)가 이벤트를 가로채어 커서의 Y 위치를 바탕으로 곡선 상의 X, Y 좌표 역계산 -> 조작부 위치 이동 및 색상 업데이트

## core 모듈과의 연동 방식
- 기존의 슬라이더 구조와 완벽히 호환되도록 만듭니다. (값 0~100 혹은 0~360을 onChange 콜백으로 넘겨주는 형태는 동일 유지)
- 단지 내부적으로 Y값 퍼센티지를 구하는 방식과 썸(Thumb)을 화면에 그리는 방식만 SVG Path 기반으로 변경됩니다.

## 상태 처리
- 사용자가 조작부를 누르고(isDragging) 마우스를 상하로 움직일 때마다 `getPointAtLength`를 이분 탐색(Binary Search)하여 가장 커서 Y값과 가까운 곡선 지점을 찾아 상태를 업데이트합니다.

## 에러 표시
- 수학적 연산 오류 방지: 슬라이더 범위를 넘어갈 경우(0 미만, 100 초과) 곡선의 양 끝단(0 또는 총 길이)에 멈추도록 예외 처리를 둡니다.

## 검증 방법
- 썸(Thumb)이 마우스 포인터의 좌우 궤적과 무관하게 곡선의 S라인을 절대 벗어나지 않는지 확인.
- S라인의 굴곡을 따라 내려갈 때 색상 스코어가 정상적으로 변화하는지 확인.

## 확장 가능성
- 향후 S자 외에도 물결(W), 원형(Circle) 등 어떠한 기하학적 형태의 패스(Path)를 주더라도 동일한 로직으로 슬라이더 작동이 가능한 범용적인 모듈로 확장할 수 있습니다.
