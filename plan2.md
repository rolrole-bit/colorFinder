# DYE MASTER 최종 결과 화면 중앙 정렬 및 가로 폭 최적화 화면 흐름 계획 (plan2.md)

## 1. UI 필요 여부
- **예**: 최종 결과 화면(`ScoreboardView.js`)의 HTML 구조 래핑 및 CSS 스타일 수정이 수반됩니다.

## 2. 사용자 흐름
1. **최종 라운드 완료**: 3라운드 색상 맞추기 완료 후 최종 결과 화면으로 자동 전환됩니다.
2. **정돈된 점수판 확인**: 사용자는 화면 전체로 방대하게 퍼져 있던 정보 대신, 화면 중앙의 최적화된 영역(`800px` 가로폭) 내에 모여 있는 본인의 닉네임, 최종 점수, 라운드별 점수 브레이크다운, 한줄평을 깔끔하게 확인합니다.
3. **가독성 향상**: 시선이 좌우 끝으로 분산되지 않고 중앙에 고정되므로 등수 변화와 랭킹 리스트를 훨씬 빠르고 인지하기 편하게 읽습니다.
4. **버튼 인터랙션**: 랭킹 카드 바로 밑에 가지런히 정렬된 "다시 하기" 및 "공유하기" 버튼을 눌러 다음 게임을 시작하거나 공유를 수행합니다.

## 3. 핵심 모듈과의 연동 방식
- `ScoreboardView.js`의 `container.innerHTML` 구조 내부에 `.magazine-content` 클래스를 가진 래퍼 `div`를 삽입하여 UI를 감싸줍니다.
- 버튼 영역의 `max-width` 인라인 스타일을 `1000px`에서 `800px`로 일괄 조정하여 콘텐츠 컨테이너 폭과 완벽히 동기화합니다.

## 4. 상태 처리
- 기존 상태 처리 로직(점수 누적, 난이도별 배율 적용, 랭킹 리스트 로드)은 그대로 작동하며 레이아웃 변경의 영향에서 완전히 격리됩니다.

## 5. 에러 표시
- 레이아웃이 좁아지더라도 랭킹 리스트의 스크롤바가 숨김 처리되어 있고 텍스트 말줄임표 처리(`text-overflow: ellipsis`)가 기 적용되어 있어, 이름이 매우 길거나 점수가 높아도 깨짐 없이 자연스럽게 렌더링됩니다.

## 6. 검증 방법
- 로컬 웹서버 구동 후 직접 테스트를 수행하여 다음 사항들을 육안 및 브라우저 개발자 도구의 엘리먼트 탭으로 검증합니다:
  - 데스크톱 전체화면 기준 콘텐츠의 최대 너비가 `800px`에 고정되어 있는지 확인.
  - 브라우저 너비를 줄였을 때 좌우 마진이 최소 `1.5rem` 유지되면서 부드럽게 줄어드는지 확인.
  - 하단 고정 버튼 영역이 콘텐츠와 어긋나지 않고 가로 위치가 알맞게 일치하는지 확인.

## 7. 확장 가능성
- 최대 너비가 `800px`로 통일되어 안정성이 증가함에 따라, 향후 랭킹 리스트 아래에 "나의 성장 그래프"나 "색상 매칭 오차율 차트" 같은 시각화 요소를 추가 배치할 때 여백 걱정 없이 쉽게 삽입할 수 있습니다.
