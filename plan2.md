# DYE MASTER 슬라이더 드래그 방향성 일치 화면/동작 흐름 계획 (plan2.md)

## 1. UI 필요 여부
- UI 레이아웃의 추가는 없으나, 사용자가 겪는 물리적 조작 드래그 방향에 따른 그래픽의 움직임을 정교하게 동기화합니다.

## 2. 사용자 흐름
1. **끌어 내리기 (Drag Down)**:
   - 사용자가 슬라이더 트랙을 터치하여 아래로 마우스를 끕니다.
   - 테이프 그래픽이 자연스럽게 마우스를 따라 아래로 이동하며, 가운데 기준선에는 이전보다 윗부분에 존재하던 색상 값들이 차례로 채워집니다.
2. **끌어 올리기 (Drag Up)**:
   - 사용자가 슬라이더 트랙을 터치하여 위로 마우스를 끕니다.
   - 테이프 그래픽이 마우스를 따라 위로 올라가며, 가운데 기준선에는 아래쪽에 대기하던 색상 값들이 노출됩니다.

## 3. 핵심 모듈과의 연동 방식
- `TapeSlider.onMove` 내의 `v += dv` 공식을 `v -= dv`로 반전시킴으로써, Y축 좌표 변화율과 테이프 렌더링 프레임의 이동 비주얼을 동일 위상으로 교정.

## 4. 상태 처리 및 연동
- 드래그 연산 상의 부호 반전은 H, S, L 모든 슬라이더에 공통 적용되어 사용자가 어느 슬라이더를 잡고 조작하더라도 일관성 있는 방향성 피드백을 받도록 보장.

## 5. 검증 방법
- 마우스 드래그를 한 번에 크게 아래로 홱 내렸을 때, 슬라이더의 띠 역시 이에 정확히 동행하여 아래로 흘러내리는지 촉각적/시각적 감각 교차 검증.

## 6. 확장 가능성
- 플레이어의 기호에 맞추어 드래그 방향을 반대로 반전시키는 옵션(Inverted Drag)이 필요할 경우, 이 부호 반전 로직의 곱셈 팩터(`* -1`) 유무로 간편하게 확장 지원할 수 있습니다.
