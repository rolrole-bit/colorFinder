# DYE MASTER H 슬라이더 보색 피킹 오류 화면/동작 흐름 계획 (plan2.md)

## 1. UI 필요 여부
- 추가적인 UI 노드 변경은 없으며, SVG 그래픽 내부의 Defs 정의(`grad-h`)만 수정합니다.

## 2. 사용자 흐름
1. **색조(Hue) 조작**: 플레이어가 가장 왼쪽의 H 슬라이더를 잡고 상하로 스크롤합니다.
2. **기준선 시각적 피킹**: 
   - 테이프를 움직여 중앙선에 초록색이 걸치면 유저 피킹 컬러도 초록색이 됩니다.
   - 중앙선에 파란색이 걸치면 유저 피킹 컬러도 파란색이 됩니다.
   - 기존의 어처구니없는 보색 불일치 현상이 완벽히 해결되어, 사용자가 시각적으로 가리키는 정확한 타겟 색조를 기분 좋게 획득할 수 있습니다.

## 3. 핵심 모듈과의 연동 방식
- SVG Defs 내 `#grad-h` 선형 그라디언트 띠 스톱 컬러 순서를 정방향 HSL 스펙트럼 배열로 재배열하여 렌더링 물리 궤적과 내부 변수 `currentH` 매핑 구조 통합.

## 4. 상태 처리 및 연동
- `currentH`는 0에서 360도로 증감하며, 변경된 정방향 그라디언트 정의에 의해 시각적 지시선 위치와 수학적 수치 위상이 정확히 일치함.

## 5. 검증 방법
- H 슬라이더 중앙에 빨간색, 노란색, 초록색, 파란색, 보라색을 각각 위치시켜 보며 실제 피킹되는 RGB 배경색과 HEX 코드가 일치하는지 순차적으로 검증.

## 6. 확장 가능성
- 차후 색상환의 기준점(예: Red가 아닌 다른 색상부터 시작)을 바꾸어야 하는 브랜딩 디자인 변경 시, `grad-h` 그라디언트 스톱 순서와 `min/max` 범위만 함께 회전시켜 주면 간편하게 수용 가능합니다.
