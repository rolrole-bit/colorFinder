# DYE MASTER 믹싱 연출 화면/동작 흐름 계획 (plan2.md)

## 1. UI 필요 여부
- 별도의 새로운 UI 뷰나 모달창은 생성하지 않습니다.
- 기존 `GameView`의 HSL 슬라이더 3개와 탑승하고 있는 돔 요소(`guessBg`, `hexDisplay`, `centerLine` 등)의 속성값을 제어하여 믹싱 연출을 표현합니다.

## 2. 사용자 흐름
1. **타겟 메모라이즈 스크린**: 5초간 원래 맞춰야 하는 타겟 컬러가 전체 화면에 띄워집니다.
2. **추측 화면 시작 (Transition to Guess)**: 메모라이즈가 끝나자마자 화면이 페이드오버 되면서 3단 슬라이더 화면이 등장합니다.
3. **믹싱 애니메이션 시작 (0.0초 ~ 0.5초)**:
   - 플레이어가 아직 조작하지 않았다면, 0.5초간 3개의 슬라이더가 댄싱하듯 위아래로 무작위 무빙을 칩니다.
   - 믹싱 속도는 일반 드래그보다 빠르고 현란하게 연출하여 "색상 섞기"가 일어나고 있음을 암시합니다.
   - 이때 탑승하고 있는 상단 라운드 텍스트, 헥스코드 박스, 중앙 기준선도 이에 맞추어 실시간으로 반짝이며 색상이 갱신됩니다.
4. **플레이어 조작 대기**: 0.5초가 지나면 믹싱이 끝나고 제자리에 정지합니다.
5. **플레이어 즉시 조작 (인터럽트)**:
   - 만약 0.2초 시점에 플레이어가 화면의 아무 슬라이더 영역(`touch-h`, `touch-s`, `touch-l`)을 터치(PointerDown)하면 그 즉시 믹싱 연출이 일시정지(Cancel)됩니다.
   - 마우스나 손가락을 뗀 것이 아니므로 터치한 슬라이더는 즉시 플레이어의 마우스/터치 드래그 위치에 맞추어 조작 모드로 넘어갑니다.

## 3. 핵심 모듈과의 연동 방식
- `GameView.js` 내의 `TapeSlider` 인스턴스인 `hueSlider`, `satSlider`, `lightSlider`의 `value` 속성을 직접 수정하고 `updateTransform()`과 `updateColor()`를 갱신합니다.
- `pointerdown`이 트리거되었을 때, `isMixing`이 `true`라면 `cancelShuffle()`을 실행하여 `requestAnimationFrame`을 취소하고 `isMixing`을 `false`로 돌립니다.

## 4. 상태 처리 및 연동
- `isMixing`: 현재 믹싱 연출이 동작 중인지 식별하는 불리언 플래그.
- `shuffleFrameId`: 믹싱에 사용되는 `requestAnimationFrame` ID.
- `shuffleStartTime`: 믹싱 연출이 시작된 타임스탬프.

## 5. 검증 방법
- 믹싱 중 다른 영역이나 화면을 눌러도 자연스럽게 중단되는지 테스트.
- 0.5초 대기 시 0.5초 내에 믹싱이 완전히 종료되고 안정적으로 일반 플레이가 가능한지 테스트.

## 6. 확장 가능성
- 차후 지옥 모드 등 난이도가 올라갈 때 믹싱되는 시간을 더 길게 하거나, 믹싱 속도를 난이도에 따라 랜덤하게 가변할 수 있도록 난이도 계수와 결합할 수 있습니다.
