HTTPS 배포 가이드

리버스 프록시 방식 (권장)

Nginx/Cloudflare 뒤에 배치 시 TRUST_PROXY=true 설정. Let's Encrypt로 무료 TLS 인증서 적용. HSTS 헤더는 Helmet이 자동 전송합니다.

# nginx 예시 server { listen 443 ssl; ssl_certificate /etc/letsencrypt/live/domain/fullchain.pem; ssl_certificate_key /etc/letsencrypt/live/domain/privkey.pem; location / { proxy_pass http://127.0.0.1:8080; proxy_set_header X-Forwarded-For $remote_addr; proxy_set_header X-Forwarded-Proto $scheme; } }

수동 설치 (install.bat 없이)

# PM2 방식 npm install -g pm2 pm2 start server/index.js --name dyemaster pm2 save pm2-startup install # NSSM 방식 (https://nssm.cc) nssm install DyeMaster "C:\Program Files\nodejs\node.exe" "E:\경로\server\index.js" nssm start DyeMaster

10시놀로지 NAS 배포 가이드

이전 버전(정적 파일)과 달리 v3.0은 Node.js 서버가 필수입니다. 아래 순서대로 진행하세요.

STEP 1. Node.js 설치

패키지 센터 → Node.js v20 설치

DSM 패키지 센터에서 "Node.js v20" 검색 후 설치. 없으면 커뮤니티 소스 추가 필요.

# SSH 접속 후 확인 ssh admin@시놀로지IP -p 포트 node -v # v20.x.x 확인 npm -v # 10.x.x 확인

STEP 2. 프로젝트 업로드

# 방법 A: Git Clone (권장) cd /volume1/web git clone https://github.com/rolrole-bit/colorFinder.git dyemaster cd dyemaster # 방법 B: File Station으로 ZIP 업로드 # /volume1/web/dyemaster/ 에 압축 해제

STEP 3. 의존성 설치 & 환경설정

# 의존성 설치 cd /volume1/web/dyemaster npm install --production # 환경변수 설정 cp .env.example .env vi .env
.env 설정 (시놀로지용)

아래 값을 시놀로지 환경에 맞게 수정:

PORT=8080 NODE_ENV=production CORS_ORIGINS=http://시놀로지IP:8080,https://도메인.synology.me TRUST_PROXY=true RATE_LIMIT_SESSION=5 RATE_LIMIT_ROUND=20 RATE_LIMIT_RANKING=30 MAX_SESSIONS=1000 MAX_RANKINGS=1000

STEP 4. PM2로 영구 실행

왜 PM2인가?

node server/index.js는 SSH 종료 시 함께 종료됩니다. PM2는 프로세스 매니저로 자동 재시작, 로그 관리, 부팅 시 자동 실행을 지원합니다.

# PM2 전역 설치 npm install -g pm2 # 서버 시작 cd /volume1/web/dyemaster pm2 start server/index.js --name dyemaster # 상태 확인 pm2 status pm2 logs dyemaster # 부팅 시 자동 시작 등록 pm2 startup pm2 save # 업데이트 후 재시작 git pull npm install --production pm2 restart dyemaster

STEP 5. DSM 리버스 프록시 설정 (HTTPS)

DSM 제어판 → 로그인 포털 → 고급 → 역방향 프록시

시놀로지 내장 리버스 프록시로 HTTPS를 무료로 적용합니다. Let's Encrypt 인증서는 DSM이 자동 관리합니다.
